### fwknop — Regras Críticas de Configuração (CMD_CYCLE)

**ATENÇÃO: Sintaxe de variável no `access.conf`** — confirmado no código-fonte `server/cmd_cycle.c`:

| Sintaxe | Válida? | Descrição |
|---------|:-------:|-----------|
| `$IP` | ✅ | IP de origem do knock (correto) |
| `$SRC` | ✅ | Sinônimo de `$IP` (correto) |
| `%IP%` | ❌ | Passa literalmente — sem substituição |
| `%SRC%` | ❌ | Passa literalmente — sem substituição |

fwknopd procura prefixo `$`, não `%`. Usar `%IP%` ou `%SRC%` resulta em `ufw allow from %IP%` literal → UFW rejeita IP inválido silenciosamente.

**ATENÇÃO: Ordem UFW — primeiro match** — `DENY 5010/tcp` precisa estar depois da regra ALLOW:
- `ufw allow from $IP to any port 5010` → appenda no **final** → nunca avaliada
- `ufw insert 1 allow from $IP to any port 5010` → insere no **início** → avaliada ANTES do DENY ✅

**ATENÇÃO: `ENABLE_CMD_EXEC` pertence SOMENTE ao `access.conf`** — em `fwknopd.conf` é ignorado com "unknown configuration parameter"

- `SSH_USER` (ex: `archaris`): conecta via SSH, executa `deploy.sh`, tem `sudo NOPASSWD`

- **NUNCA criar `userlist.txt` com usuários de aplicação** — autenticação dinâmica via `auth_query`

### Volumes Docker — Princípio VII
- Sempre usar volumes bind-named externos (`driver=local, type=none, o=bind`)
- `docker volume create` deve ser executado via `run_as_deploy_user()`
- Diretórios do host criados antes de `docker compose up`

### Ownership dos arquivos em BASE_PATH
- `BASE_PATH/` e subdiretórios gerenciados → `docker_user:docker_user`
- `postgres_data/`, `postgres_logs/` → UID `999` (postgres no Alpine)

- `.env` → `chmod 644` (SSH_USER precisa fazer `source` ao executar o script)

### self-copy guard no deploy.sh
- Sempre usar `[[ $(realpath src) != $(realpath dst) ]]` antes de `cp` em `publish_compose()` e `copy_configs()`

## Movimentação de Arquivos em Lote (obrigatório)

Para mover/copiar/renomear **3 ou mais arquivos**, NUNCA usar `run_in_terminal` com `mv` repetido ou loop shell.

**Padrão obrigatório:** JSON com lista de operações + `pylanceRunCodeSnippet` Python:

```python
import shutil
from pathlib import Path

root = Path("/abs/path/to/project")
ops = [
    {"name": "arquivo.md", "from": "origem/", "to": "destino/"},
    {"name": "outro.yml",  "from": "src/",    "to": "dst/"},
]
for op in ops:
    src = root / op["from"] / op["name"]
    dst = root / op["to"]  / op["name"]
    dst.parent.mkdir(parents=True, exist_ok=True)
    if src.exists():
        shutil.move(str(src), str(dst))
        print(f"✅ {op['name']}")
    else:
        print(f"⚠️  não encontrado: {src}")
```

- 1-2 arquivos → `run_in_terminal mv` aceitável
- **3+ arquivos → Python + JSON obrigatório**
- Variações: `shutil.copy2` para copiar, `Path.unlink()` para deletar

**Ref:** `.copilot-strict-enforcement.md` REGRA 0.B

### Troubleshooting Conhecido (2026-03-12 atualizado)
- **VictoriaMetrics:** `indexdb/` pode ser criado com `root:root` → `chown -R nobody:nogroup /data/victoria-metrics`
- **Grafana + secrets:** Secret files precisam de `chmod 644` → `chmod 644 .secrets/*.txt`
- **PushGateway v1.6.2:** `--push.ttl` NÃO suportado (usar apenas v1.9+)
- **VictoriaMetrics sem porta exposta:** Não tem porta exposta no docker-compose. Para queries diretas, acessar via `enterprise-grafana` container: `docker exec enterprise-grafana wget -qO- 'http://victoriametrics:8428/api/v1/query?query=...'`
- **VictoriaMetrics remote_write:** `remote_write` está comentado em `src/config/prometheus.yaml:382` — métricas não estão sendo replicadas para o VM. Para ativar, remover comentário e reiniciar Prometheus.
- **VictoriaMetrics coleta por data:** Script disponível em `scripts/victoria_metrics_collector.py`. Executa via SSH usando `docker exec enterprise-grafana wget` para acessar `victoriametrics:8428`. Testado: 2.615 métricas, 30.496 pontos, 6 dias.
- **AlertManager DNS:** O nome do serviço Docker é `alertmanager`, NÃO `enterprise-alertmanager`. Usar em `prometheus.yaml`: `alertmanagers: [{static_configs: [{targets: ['alertmanager:9093']}]}]`
- **AlertManager — manter PARADO até stack maduro:** O AlertManager deve permanecer parado (`docker compose stop alertmanager`) enquanto a configuração do stack ainda estiver em desenvolvimento/ajuste. Isso evita envios de e-mail desnecessários durante testes e iterações. Só subir o serviço quando toda a configuração estiver estável e revisada.
- **Collector-api wf001:** Container em `/opt/docker_user/n8n-monitoring-local/`. **SEMPRE** iniciar/reiniciar via `docker compose up -d` (nunca `docker run`). Se iniciado externamente, o .env NÃO é carregado → credenciais ausentes → `Access denied (using password: NO)`.
- **API key timing attack:** Comparar API keys com `hmac.compare_digest()` nunca com `==` (OWASP A07).
- **Dashboards DS_PROMETHEUS:** UIDs de datasource como `${DS_PROMETHEUS}` precisam ser substituídos pelo UID real `prometheus` (ou o uid configurado no Grafana) em todos os painéis JSON dos dashboards.
- **node_exporter + `prometheus-node-exporter` (apt):** Se o servidor tiver o pacote apt instalado, ele ocupa a porta 9100, impedindo nosso `node-exporter.service`. O role Ansible agora para/desabilita `prometheus-node-exporter.service` antes de iniciar o nosso (task: `failed_when: false` para idempotência). Ver BUG-09.
- **node_exporter + `ProtectSystem=strict`:** `--collector.processes` requer `CAP_SYS_PTRACE` para acessar `/proc/<pid>/` de outros usuários. Incompatível com `ProtectSystem=strict` sem capability explícita. Removido dos defaults. Ver BUG-08.
- **node_exporter + estado `activating`:** Script Python de validação usa `retries=6, delay=3.0s` antes de declarar falha no check de serviço. Evita falso negativo em reinícios. Ver BUG-07.
- **VictoriaMetrics + maxLabelsPerTimeseries:** Padrão é 30. Stack usa `--maxLabelsPerTimeseries=50` (configurado em docker-compose.yaml). Warnings `dropping labels` indicam que o limite foi atingido.
